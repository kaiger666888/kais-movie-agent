### E2E Flow Traces

**Flow A: Bad-case memory across runs (B5-01/02/03)**

1. cloud-production runs ShotParallelScheduler.runWithRetry with `blacklist` option
2. Per-shot `blacklist.check()` → hit/miss/disabled/degraded
3. 'hit' → result `{status:'blacklisted', blacklist_skipped:true}`, taskFn never called
4. Permanent failures → `blacklist.record()` → compute embedding → write to AssetBus 'failed-shots' envelope (deduped)
5. Next run loads 'failed-shots' on next check() — semantic match via 0.92 cosine similarity
6. Status: **COMPLETE** (verified by B5-03 three-state distinguishable test)

**Flow B: Audio-visual sync submission (A2-01/02/03)**

1. voice phase writes voice-timeline.json (v2.0)
2. cloud-production reads `bus.read('voice-timeline')` via `_extractVoiceEntries`
3. Shots with dialogue but missing voice-timeline → throws (timing lock)
4. `getOmniReferencePack(audioRefs)` builds `@Audio{n}` prompt bindings + `hasAudio:true`
5. Pre-submit gate: hasAudio && !bindings.includes('@Audio') → throws (Pitfalls 陷阱 1 defense)
6. `submitTask({audio_refs, prompt_audio_bindings, generate_audio:true})` to gold-team
7. Status: **COMPLETE** (operator-side: actual Seedance audio_refs API contract verification deferred W-v3-3)

**Flow C: Creative history downstream lineage (B4-03/04/05)**

1. cloud-production completes videos
2. Reads envelopes of sts/char/scene → collects 3 source content_hashes
3. For each completed video → `tracker.stamp({asset_slot:'final-shots', source_hashes, content_hash})`
4. stamp() appends record to creative-history slot (atomic via AssetBus)
5. Later: `tracker.findAffected(changedHash)` reverse-BFS over Map<source_hash, Set<record>>
6. Status: **COMPLETE** (perf 0.47ms at 1000-asset scale, 1000× margin)

**Flow D: Fine-tuning manifest + human gate (B6-01/03/04/06)**

1. Pipeline accumulates failed-shots via BlacklistEngine
2. delivery handler (when `config.finetune.auto_generate=true`) calls `etl.generateManifest()`
3. generateManifest reads failed-shots, enriches with anchor/audio/recommended_action
4. PII scrubber flags suspicious metadata; poisoning detection marks outlier samples
5. Writes pending-review samples to `.pipeline-assets/finetune-pending/`
6. Operator CLI: `bin/finetune-review.js approve <id> --copyright X --pii true --label true --reviewer Y`
7. approveSample enforces 4 fields hard (throws on missing/invalid)
8. Approved → appendLine('finetune-dataset', ...); Rejected → move to rejected/
9. Later: `submitTrainingJob()` → gold-team `{task_type:'lora_training'}`
10. Status: **COMPLETE** (operator-side: populate regression-baseline.json with real prompts W-v3-5)
