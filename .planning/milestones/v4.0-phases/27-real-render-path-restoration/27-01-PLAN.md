<objective>
Fix the silent-failure bug in the motion-preview handler's Blender render submission path.

Purpose: The motion-preview handler currently calls `gtClient.submitTask({ task_type: 'blender_render', ... })` (snake_case), but `GoldTeamClient.submitTask` destructures `{ taskType }` (camelCase) — see `lib/gold-team-client.js:53`. The snake_case arg becomes `undefined`, the request body loses `task_type`, and the gold-team service silently rejects/ignores the task. Additionally, line 1078 reads `task.task_id` but the return shape is `{ taskId }` — so even if submitTask succeeded, the recorded taskId would be `undefined`. This plan closes PIPE-RENDER-01.

Output: Surgical 2-line fix (line 1074 + line 1078) plus a regression test that mocks GoldTeamClient.submitTask and asserts camelCase invocation.
</objective>
