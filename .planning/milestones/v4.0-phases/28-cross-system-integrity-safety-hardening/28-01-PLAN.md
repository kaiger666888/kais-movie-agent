UPDATE')`) to kais-aigc-platform HTTP API (`/api/canvas/v2/save-v2`), eliminating the canvas double-write race where kais-aigc-platform truncates content→150 / signature_shot→200 and overwrites our repo's complete data.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| kais-movie-agent → kais-aigc-platform HTTP | Outbound HTTP write crosses trust boundary; platform may be unreachable, slow, or return errors. |
| Local process → filesystem (sqlite read) | loadGraph still reads sqlite directly — same-trust local read, no boundary crossing. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-28-01 | Tampering | canvas-content-sync saveGraph HTTP write | mitigate | Migrate to platform HTTP API (single write path) — eliminates last-write-wins race. Per D-PIPE-INTEGRITY-01. |
| T-28-02 | Denial of Service | saveGraph when platform unreachable | mitigate | Degrade-on-unreachable (console.warn, no throw) — pipeline continues. Per D-PIPE-INTEGRITY-01. |
| T-28-03 | Information Disclosure | console.warn message may leak internal URLs/errors | accept | Warn messages are operator-facing logs on local process; low risk, no PII. Matches Phase 26/27 degrade-warn convention. |
| T-28-04 | Repudiation | saveGraph degrade silently skips write | mitigate | console.warn marker `[canvas-sync] HTTP API unreachable` is greppable in logs for audit. |
</threat_model>
