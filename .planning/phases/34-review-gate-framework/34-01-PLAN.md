## Task 2: Implement gate.py (GREEN gate)

Create `plugins/review_gates/gate.py` (~200-280 LOC) per PATTERNS.md "gate.py Anatomy" skeleton.

**Required content:**

1. **Module docstring** referencing Phase 34-01, SC#1, CF-01 (3 modes), CF-05 (max_retries).

2. **Enums:**
   ```python
   class GateMode(str, enum.Enum):
       BLOCKING = "blocking"
       WEBHOOK = "webhook"
       POLLING = "polling"

   class GateStatus(str, enum.Enum):
       PENDING = "pending"
       APPROVED = "approved"
       REJECTED = "rejected"
       CONTESTED = "contested"
       TIMED_OUT = "timed_out"
       FAILED = "failed"
   ```

3. **Exceptions:**
   - `class GateError(Exception)` — transient (network, state mismatch, invalid decision).
   - `class GateMaxRetriesExceeded(Exception)` — terminal; constructor takes `(gate_id, attempts, max_retries)`; message must start with `"CONSISTENCY_BLOCKED: "` (PIPE-GUARD-01 marker so callers/grep can identify).

4. **`@dataclass(frozen=True) GateConfig`** — required fields: `gate_id: str`, `phase: str`, `asset_bus_slots_to_lock: tuple[str, ...]`, `reviewer_role: str`. Defaults: `timeout_sec: int = 3600`, `callback_url: Optional[str] = None`, `max_retries: int = 2`, `backoff_sec: int = 300`, `default_mode: GateMode = GateMode.BLOCKING`.

5. **`@dataclass Gate`** (mutable) — fields: `config: GateConfig`, `episode_id: str`, `mode: GateMode`, plus mutable state `attempt: int = 0`, `status: GateStatus = PENDING`, `review_id`, `submitted_at`, `resolved_at`, `decision`, `suggested_action`, and `_event: threading.Event = field(default_factory=threading.Event, repr=False)`.

6. **Methods** (per PATTERNS.md):
   - `submit(self, payload: dict, *, review_client: Any = None) -> dict` — increment attempt; if `attempt > config.max_retries`: set status=FAILED + raise `GateMaxRetriesExceeded(gate_id, attempt, max_retries)`. Set `submitted_at` ISO timestamp, status=PENDING, clear Event. Return submission record dict.
   - `wait(self, timeout_sec: Optional[int] = None) -> dict` — dispatch on `self.mode`:
     - BLOCKING → `self._event.wait(timeout=effective_timeout)`; if False → status=TIMED_OUT. Return `_outcome_record()`.
     - WEBHOOK → return immediately `{"status": "awaiting_callback", "review_id": self.review_id}` (non-blocking — caller persists and exits).
     - POLLING → raise `GateError("POLLING mode wait requires runner_hooks.poll_until_terminal(); use Gate.poll_step() in a loop instead.")`.
   - `resolve(self, decision: str, suggested_action: Optional[str] = None) -> dict` — validate decision in `{approve, reject, contest}` (else GateError); set status (APPROVED/REJECTED/CONTESTED); set decision+suggested_action+resolved_at; `self._event.set()` to wake blocking waiters. Return `_outcome_record()`.
   - `_outcome_record(self) -> dict` — return the CF-04 outcome record dict.

7. **Imports:** stdlib only — `enum`, `logging`, `threading`, `time`, `dataclasses`, `datetime`, `pathlib`, `json`, `typing`. **NO** httpx, jwt, yaml, or any plugins.* import (gate.py is leaf module).
