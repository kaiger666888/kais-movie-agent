# Plan 34-03: Runner Hooks Adapter (delegate_task Integration)

<objective>
Implement `plugins/review_gates/runner_hooks.py` — the adapter between Phase 35
orchestration runner and Phase 34 gate framework. Three entry points:

1. `pause_for_review(gate_id, episode_id, payload)` — submit + write `awaiting_review` state.
2. `resume_from_callback(body, signature, timestamp)` — HMAC verify → resolve → write outcome.
3. `poll_until_terminal(gate_id, timeout_sec)` — polling-mode wait loop.

Plus `mark_episode_failed()` for PIPE-GUARD-01 episode-fail semantics.

Scope: adapters ONLY. State machine in 34-01; YAML config in 34-02. This module
imports from both + reuses Phase 32 `review_platform.ReviewPlatformClient` and
Phase 33 `PipelineStateStore` + `AssetBus`.
</objective>

## Task 2: Implement runner_hooks.py (GREEN gate)

Create `plugins/review_gates/runner_hooks.py` (~200-280 LOC) per PATTERNS.md "runner_hooks.py Anatomy" skeleton.

**Required content:**

1. **Module docstring** referencing Phase 34-03, SC#3/4/5, CF-03/04/05.

2. **Imports** (per PATTERNS.md):
   ```python
   import json
   import logging
   import os
   import time
   from datetime import datetime, timezone
   from typing import Any, Optional

   from plugins.kais_aigc.review_platform import ReviewPlatformClient
   from plugins.pipeline_state.asset_bus import AssetBus
   from plugins.pipeline_state.store import PipelineStateStore
   from plugins.review_gates.gate import (
       Gate, GateConfig, GateError, GateMaxRetriesExceeded, GateMode, GateStatus,
   )
   from plugins.review_gates.gate_config import GATE_REGISTRY, to_gate_config
   ```

3. **Module-level factory helpers** (mirror Phase 33-04 `_state_store()` / `_asset_bus()` pattern):
   ```python
   def _review_client() -> ReviewPlatformClient:
       """Construct or return cached ReviewPlatformClient (Phase 32)."""
       # Tests monkeypatch.setattr this to inject MagicMock.
       return ReviewPlatformClient()

   def _asset_bus(workdir: str | None = None) -> AssetBus:
       return AssetBus(workdir or os.getcwd())

   def _state_store(workdir: str | None = None) -> PipelineStateStore:
       return PipelineStateStore(workdir or os.getcwd())
   ```

4. **`pause_for_review(gate_id, episode_id, payload, *, mode=None) -> dict`**:
   - Look up `GateConfig` via `to_gate_config(gate_id)`.
   - Construct `Gate(config=config, episode_id=episode_id, mode=mode or config.default_mode)`.
   - Call `gate.submit(payload)` — this raises `GateMaxRetriesExceeded` if attempt > max_retries. **Let it propagate** (do not swallow).
   - Call `_review_client().submit_review(type=gate_id, content_ref=f"{episode_id}/{config.phase}", callback_url=config.callback_url)`.
   - If result has `"degraded": True` → auto-resolve approve (mirrors review_platform DEGRADED_AUTO).
   - Else set `gate.review_id = result["review_id"]`.
   - Write `awaiting_review` to PipelineState via `_state_store()`: `state.phases[config.phase] = {"status": "awaiting_review", "review_id": gate.review_id, "submitted_at": gate.submitted_at}`. Save.
   - Return `{gate_id, episode_id, review_id, status: gate.status.value, attempt: gate.attempt}`.

5. **`resume_from_callback(body: str, signature: str, timestamp: int) -> dict`**:
   - Call `_review_client().verify_callback(body, signature, timestamp)`. If False → raise `PermissionError("Invalid HMAC callback signature")`.
   - Parse `callback_data = json.loads(body)`; extract `gate_id`, `decision`, `suggested_action`.
   - Look up pending gate: in the simple case, reconstruct from PipelineState (`phases[phase].status == "awaiting_review"`). For Phase 34 scope, a single in-memory `_PENDING_GATES: dict[str, Gate]` module-level cache is acceptable (Phase 35 runner will manage persistence properly; this is the adapter shim).
   - Call `gate.resolve(decision, suggested_action)` → outcome record.
   - Write outcome to asset bus via `_write_review_outcome(gate, outcome)`.
   - Update PipelineState: `phases[phase].status = "approved"` (if approve) or `"rejected"` (if reject).
   - If decision == "reject" and suggested_action: include `{"rollback_to": suggested_action}` in return dict so Phase 35 runner can jump back.
   - Return outcome dict (+ rollback_to if applicable).

6. **`_write_review_outcome(gate: Gate, outcome: dict) -> None`**:
   - `bus = _asset_bus()`
   - `current = bus.read("review-outcomes") or {"outcomes": [], "version": 1}`
   - `current["outcomes"].append(outcome)`
   - `bus.write("review-outcomes", current, envelope=True)` — atomic write via Phase 33 AssetBus.

7. **`poll_until_terminal(gate_id: str, timeout_sec: int, *, interval_sec: int = 30) -> dict`**:
   - Look up GateConfig; construct Gate.
   - Call `gate.submit({})` to register attempt (may raise GateMaxRetriesExceeded).
   - Loop: while `elapsed < timeout_sec`: call `_review_client().query_review_status(gate.review_id)`; if `state in {"resolved", "closed"}`: break and resolve. Else `time.sleep(interval_sec)`.
   - On timeout: return `{status: "timed_out", ...}`.
   - Write outcome + update state (same as resume_from_callback).

8. **`mark_episode_failed(episode_id: str, gate_id: str, exc: GateMaxRetriesExceeded) -> None`** (PIPE-GUARD-01):
   - `store = _state_store()`
   - `state = store.load()`
   - `phase = to_gate_config(gate_id).phase`
   - `state.phases[phase] = {"status": "failed", "failed_at": datetime.now(timezone.utc).isoformat(), "error": str(exc)}` — `str(exc)` already starts with `"CONSISTENCY_BLOCKED: "` from Plan 34-01.
   - `store.save(state)`.
   - Log ERROR level.
