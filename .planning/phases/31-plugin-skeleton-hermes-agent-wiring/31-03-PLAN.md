Add per-plugin smoke tests that verify ROADMAP SC#3 ("三个 plugin 的 entry module 可被 `python -c \"import ...\"` smoke-import") and exercise each plugin's tool surface end-to-end with stub handlers.

Purpose: Plan 31-01 created the artifacts. Plan 31-02 proved the real loader can discover them. This plan adds the *fast, isolated, per-plugin* smoke tests that Phase 32/33/34 will run after every change to make sure they haven't broken the skeleton contract. These tests are intentionally **decoupled from the real PluginManager** — they use a fake ctx (the pattern from PATTERNS.md "Test Pattern") so they run in <1 second and don't depend on config state.

**Why both loader tests (31-02) AND smoke tests (31-03)?** Different failure modes:
- Loader tests catch "plugin won't be discovered by hermes-agent in production" — a manifest format problem.
- Smoke tests catch "register(ctx) produces the wrong tool surface" or "stub handler crashes" — a code contract problem.
Phase 32+ will iterate on handler bodies; smoke tests catch regressions in the tool surface contract immediately.

Output: 3 smoke test files (one per plugin). The `tests/__init__.py` package markers were created in Plan 31-02.
</objective>
