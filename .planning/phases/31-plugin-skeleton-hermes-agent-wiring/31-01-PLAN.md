**Critical corrections from the original phase_context (researched — see CONTEXT.md CRITICAL-FINDING-01/02):**
1. Manifest is `plugin.yaml` (YAML), NOT `plugin.json`. The hermes-agent loader only scans for `plugin.yaml` then `plugin.yml`.
2. `register(ctx)` lives in `__init__.py`, NOT in a separate `client.py`/`state.py`/`gates.py` entry module. Implementation modules get added in later phases.
3. Plugin `kind: standalone` (opt-in via `plugins.enabled`), NOT `kind: backend` (which is for pluggable providers of existing core tools like image_generate — wrong plugin type here).

Loader contract for the new plugins:
1. Each plugin directory MUST contain `plugin.yaml` (loader checks this filename first)
2. Each plugin directory MUST contain `__init__.py` (loader raises FileNotFoundError otherwise)
3. `__init__.py` MUST define `register(ctx)` callable (loader calls `register_fn(ctx)`)
4. The plugin becomes importable as `hermes_plugins.<slug>` where slug = key.replace("/","__").replace("-","_")
5. For our plugins: hermes_plugins.kais_aigc / hermes_plugins.pipeline_state / hermes_plugins.review_gates
