<objective>
Reorder the PHASES array so spatio-temporal-script runs before scene-generation, restoring the real data dependency.

Purpose: scene-generation (stageOrder 8) calls `bus.read('spatio-temporal-script')` at lib/phases/index.js:2564, but spatio-temporal-script (stageOrder 10) writes the asset at lib/phases/index.js:885. In the default V6 run order, sts has NOT yet executed when scene-generation runs → the read returns null → scene-generation falls through to a single hardcoded default scene (lines 2573-2580). This silently degrades all scene data to a 1-scene placeholder and starves downstream composition/render of real scene structure.

Per user decision (CONTEXT.md PIPE-DATA-02, LOCKED): reorder via stageOrder + array position. New order: spatio-temporal-script (8) → scene-generation (9) → scene-selection (10). No stub sts path (would obscure data flow). No hard fail (would break degraded mode runnability).

Output: Patched PHASES array + synced VALID_PHASES + unit/integration tests proving the new ordering.

CRITICAL IMPLEMENTATION DETAIL: pipeline execution iterates the PHASES array by INDEX (lib/pipeline.js:659 `for (let i = startIdx; i < PHASES.length; i++)`), NOT by stageOrder field. So BOTH must change: array declaration order AND the stageOrder numeric field. The stageOrder field feeds canvas positioning (lib/canvas-sync-hook.js:142-150), reporting (lib/pipeline.js:256, 323, 698), and Hermes audit. Leaving stageOrder stale while reordering the array would create silent drift between execution order and reported order.

CRITICAL EXPORTS DETAIL (verified during checker revision): `PHASES` (lib/pipeline.js:50) is declared `const PHASES = [...]` and is MODULE-PRIVATE — it is NOT a named export. The only export from lib/pipeline.js is `export default Pipeline;` (line 704). Access PHASES via the static accessor `Pipeline.getPhases()` (lib/pipeline.js:701). Likewise `VALID_PHASES` (lib/hermes-client.js:16) is MODULE-PRIVATE. Tests that need to read VALID_PHASES must parse the source textually (grep/regex) rather than import it.
